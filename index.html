Responsive footer columns without explicit breakpoints or JavaScript

When building a component for an unknown amount of content, such as components with slots or a footer component with infinite possibilities of links per column, it's fundamentally required your containers are responsive without explicit breakpoints or JavaScript because you don't know when or where to make a breakpoint (unknown amounts of content) and JavaScript shouldn't be necessary for dynamic, responsive containers.

Experiments

  1. Flexbox rows with wrapping
  2. Flexbow columns with wrapping
  3. Grid with column flow
  4. Grid with row flow

Conditions

  1. Each container works alongside other containers (responsiveness)
  2. Each row's columns are of the same height
  3. Text-based children do not wrap
  4. Mobile responsiveness without explicit breakpoints or JS
  5. Containers support 50+ items

Results

  • Flex rows do work: even with set widths of the child elements, the container will grows left to right, which doesn't work when you only have a few items in a column (which you want to grow vertically)
  • Flex columns cannot work because the flex container with column direction and wrapping will not expand to contain wrapped columns, which is a huge overflow problem
  • Grid with column flow is similar to grid with row flow - you must have a predefined max-height for the container to break into new columns automatically, which means managing an unknown number of items results in an impossible scenario of not know when to set mobile breakpoints
